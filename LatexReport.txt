\section*{Abstract}
Financial transaction fraud represents a multi-billion dollar threat to the stability of the global digital economy. Traditional detection paradigms, heavily reliant on static rule-based validation or aggregated classical machine learning classifiers, fail to capture the complex, non-linear chronological context embedded within continuous transaction streams. This research addresses these limitations by designing, training, and deploying a robust end-to-end operational software and intelligence framework driven by a deep \texttt{TransformerClassifier}. Utilizing multi-head self-attention mechanisms, the proposed neural network processes transactional attributes as sequential elements, implicitly discovering hidden behavioral anomalies without the need for manual sliding-window feature engineering.

To handle the extreme data imbalance typical of real-world credit card logs, a data engineering pipeline incorporating localized cyclic temporal decomposition, mathematical log-stabilization, and structured random under-sampling is developed. The optimized model is subsequently exported into a pre-compiled Open Neural Network Exchange (\texttt{ONNX}) continuous framework and embedded directly within a Spring Boot backend as an in-memory execution session using the local ONNX Runtime Java API. A presentation layer is constructed using Vite, ReactJS, and TypeScript to deliver real-time reactive visual telemetry to security analysts. 

Empirical validation on a large-scale dataset spanning over $555,719$ transaction sequences confirms that the model achieves an exceptional anomaly capture sensitivity, recording a critical fraud Recall rate of $96.88\%$, a stable minority Precision of $0.13$, and a robust Precision-Recall Area Under the Curve (PR-AUC) of $0.8026$. The system demonstrates high runtime efficiency, maintaining sub-millisecond inference latencies while eliminating external network serialization overhead. This report details the underlying mathematical concepts, full-stack architectural design, concrete integration code, and experimental results, establishing a scalable roadmap for production-grade financial fraud mitigation.

\section{Introduction}

\subsection{Context and Motivation}
In the contemporary financial ecosystem, credit and debit cards have become the primary method for consumer payments in the United States, making every transaction a potential opportunity for exploitation. Financial activity has shifted from, according to Security.org\cite{Security.org}, localized events to a continuous digital stream where 61.3 million Americans—roughly one in three adults—experienced fraudulent charges in the past year alone. While this digital shift offers convenience, it has enabled sophisticated threat actors to execute remote maneuvers such as data breaches, skimming, and credential theft, which now account for 95 percent of all fraud cases, while physical card theft has dwindled to just 5 percent.

Historically, banking institutions protected their assets against a diverse array of threats—including credit card fraud, loan fraud, and money laundering—using static rule-based engines\cite{Hasnain Kashif, & Fawad Naseer}
. These legacy structures cross-reference incoming transaction metadata against fixed conditional thresholds. However, recent analysis indicates these rigid systems achieve only about $78.3\%$ detection accuracy, struggling to adapt as financial fraud, waste, and abuse escalate to a global cost of $\$5.4$ trillion annually
.

\subsection{Significance and Role of the Project}
This internship project bridges the gap between advanced self-attention deep learning research and practical, high-performance software engineering within corporate banking environments. The overarching role of the developed system is to introduce an automated, intelligent risk mitigation layer that functions concurrently with core transactional accounting loops without introducing execution blocks or latency penalties. 

From an AI research perspective, the significance of this project lies in its utilization of the Transformer architecture to exploit sequential self-attention mechanisms over tabular data. In financial logs, transaction attributes (such as timestamps, merchant categories, and purchase volumes) possess deep non-linear relationships that traditional models fail to map without extensive, manual, and expensive feature engineering. By tokenizing these tabular features into aligned temporal sequences, the model automatically weighs the relative significance of past events against the current payment request. From an operational engineering perspective, the project demonstrates how complex deep learning checkpoints can be deployed within an enterprise Spring Boot ecosystem using native in-memory binaries via an embedded Open Neural Network Exchange (\texttt{ONNX}) runtime, bypassing the high latency overhead typically caused by external Python-based REST microservices.

The academic and industrial engineering landscape of anomaly detection in financial records covers a wide spectrum of computational intelligence, soft computing, and machine learning methodologies, as summarized in Table \ref{tab:related_works_matrix}. Early soft-computing paradigms leveraged biology-inspired frameworks such as the Artificial Immune System (AIS) and Genetic Algorithms (GA) to exploit structural self-organization and cost-effective, rapid execution; however, these approaches inherently required extensive training environments or highly specialized domain knowledge to operate. To model probabilistic states and temporal configurations, architectures like Bayesian Belief Networks (BBN) and Hidden Markov Models (HMM) were introduced, but they frequently exhibited low accuracy thresholds and severe performance degradation when processing massive, enterprise-grade data streams.

Within the classical supervised machine learning domain, shallow classifiers such as Decision Trees (DT) and Support Vector Machines (SVM) offered high adaptability and structural resilience to biased training samples, yet they introduced significant computational latency due to isolated conditional checks and suffered from poor scalability. To streamline high-dimensional data retrieval, unsupervised paradigms including standard Clustering Methods (CN) and Self-Organizing Maps (SOM) were deployed to map complex transaction spaces into simpler, interpretable representations, though they regularly suffered from elevated false-positive rates when clean, real-world data profiles were unavailable. To overcome these localized constraints, standard Neural Networks (NN) emerged to deliver superior accuracy and processing velocity, establishing the structural baseline for modern deep-learning anomaly detection frameworks.
\subsection{Objectives}

Directly addressing the architectural bottlenecks, scalability constraints, and elevated false-positive profiles inherent in classical soft-computing and shallow machine learning paradigms detailed above, the core mission of this project is to architect, evaluate, and deploy a sequence-aware, end-to-end deep learning fraud detection framework. To systematically transcend the limitations of prior arts, the project fulfills the following specific technical objectives:

\begin{itemize}
    \item \textbf{Data Engineering Optimization and Imbalance Mitigation}: Overcome the data format sensitivities and high false-positive risks typical of clustering methods by building a robust, scalable ingestion pipeline. This pipeline will clean, normalize, and transform massive, highly skewed multi-year financial logs into balanced, standardized sequence tokens tailored for deep-learning ingestion.
    
    \item \textbf{Sequence-Aware Deep Learning Architecture Formulation}: Address the structural incapacity of Hidden Markov Models (HMM) and Bayesian Networks in processing streams with high precision. This objective focuses on constructing a customized PyTorch Transformer model that leverages multi-head self-attention mechanisms to seamlessly capture multi-modal, long-range dependencies across numerical and categorical transaction vectors simultaneously.
    
    \item \textbf{High-Performance, Low-Latency Enterprise Integration}: Direct combat the computational latencies, isolated conditional checks, and poor large-scale scalability observed in traditional Decision Trees and Support Vector Machines (SVM). By compiling the trained Transformer model into an optimized ONNX format and embedding it directly within a production-grade Spring Boot backend via the ONNX Runtime Java API, the framework achieves sub-millisecond, in-memory inference without external network overhead.
    
    \item \textbf{Interactive Analytics and Interpretable Monitoring Presentation}: Resolve the interpretability constraints and complex, opaque mappings associated with early connectionist and unsupervised topologies like Self-Organizing Maps (SOM). This is achieved by implementing a responsive, TypeScript-backed React dashboard designed to translate complex model outputs into real-time visual transaction distributions, intuitive user risk stratification tiers, and immediate security alerts.
\end{itemize}

\subsection{Scope of the Project}
This research and development work is executed over the comprehensive \textbf{Kaggle: Credit Card Transactions Fraud Detection Dataset}\cite{dataset}, encompassing two full years of continuous synthetic financial operations. The project scope is structurally bounded by the following technical limits:
\begin{enumerate}
    \item \textbf{Algorithmic Scope}: Modeling is limited to tabular sequence classification using multi-head self-attention, omitting graph-based neural structures (GCNs) or raw image-based processing.

    \item \textbf{Operational Scope}: Data balancing is constrained to offline structural random under-sampling at the training partition stage, setting up a performance baseline for future cost-sensitive optimizations.
\end{enumerate}

\section{Mathematical Foundations and Modeling Frameworks}

\subsection{Architectural Frameworks and Mathematical Specifications}
To evaluate the predictive capabilities of deep sequential architectures against traditional parametric and ensemble machine learning approaches under severe class imbalance, four distinct modeling methodologies are formalized below.

\subsubsection{Logistic Regression }\cite{LR}
As a foundational baseline, Logistic Regression models the probability of fraud by assuming a linear relationship between the log-odds of the binary target variable and the input features. Let $\mathbf{x} \in \mathbb{R}^{d}$ represent a flattened, non-sequential feature vector extracted from a single transaction event. The model maps this vector to a continuous risk probability $P(y=1|\mathbf{x})$ via a linear combination scaled by a parameter weight vector $\mathbf{w} \in \mathbb{R}^{d}$ and an operational bias $b \in \mathbb{R}$:
\begin{equation}
z = \mathbf{w}^T \mathbf{x} + b
\end{equation}
This unnormalized logit score $z$ is then transformed into a bounded probability space using the standard logistic sigmoid activation operator $\sigma(z)$:
\begin{equation}
P(y=1|\mathbf{x}) = \sigma(z) = \frac{1}{1 + e^{-(\mathbf{w}^T \mathbf{x} + b)}}
\label{eq:logistic_regression}
\end{equation}
While computationally efficient and highly interpretable via exponentiated coefficients, Logistic Regression is fundamentally restricted by its linear decision boundary. It cannot natively capture high-order non-linear feature interactions without extensive manual engineering.

\subsubsection{Decision Tree }\cite{DT}
As a step toward capturing non-linear boundaries, a single Decision Tree operates as a non-parametric model that recursively partitions the input feature space $\mathbb{R}^d$ into orthogonal hyper-rectangles. At each parent node containing a subset of training data $\mathcal{D}$, the algorithm evaluates possible binary splits defined by a feature dimension $j \in \{1, 2, \dots, d\}$ and a split threshold $s \in \mathbb{R}$:
\begin{equation}
\mathcal{D}_L(j, s) = \{ (\mathbf{x}, y) \in \mathcal{D} \mid x_j \le s \} \quad \text{and} \quad \mathcal{D}_R(j, s) = \{ (\mathbf{x}, y) \in \mathcal{D} \mid x_j > s \}
\end{equation}
The optimal parameter split pair $(j^*, s^*)$ is selected to maximize the reduction in impurity (impurity gain). The baseline model utilizes Gini Impurity $I_G(\mathcal{D})$ as the optimization criterion:
\begin{equation}
I_G(\mathcal{D}) = 1 - \sum_{k \in \{0, 1\}} p_k^2
\end{equation}
where $p_k = \frac{1}{|\mathcal{D}|} \sum_{(\mathbf{x}, y) \in \mathcal{D}} \mathbb{I}(y = k)$ represents the empirical probability of class $k$ within node $\mathcal{D}$. The split selection solves:
\begin{equation}
(j^*, s^*) = \arg\max_{j, s} \left[ I_G(\mathcal{D}) - \left( \frac{|\mathcal{D}_L(j,s)|}{|\mathcal{D}|} I_G(\mathcal{D}_L(j,s)) + \frac{|\mathcal{D}_R(j,s)|}{|\mathcal{D}|} I_G(\mathcal{D}_R(j,s)) \right) \right]
\end{equation}
Recursive partitioning is executed until reaching stopping criteria (e.g., maximum tree depth or minimum node size thresholds), at which point leaf nodes output the class distribution probability. While highly interpretable and capable of discovering arbitrary non-linear boundaries, individual decision trees exhibit high variance and are structurally sensitive to minor perturbations in the training dataset.

\subsubsection{Random Forest }\cite{RF}
To capture complex non-linear boundaries without manual feature interactions, Random Forest operates as a non-parametric ensemble of independent Decision Trees. It relies on the principle of Bootstrap Aggregation (Bagging) combined with the random subspace method.

Let $M$ define the total number of decision trees trained within the forest ensemble. For each tree $m \in \{1, 2, \dots, M\}$, a bootstrap sample $\mathcal{D}_m$ is drawn with replacement from the global training dataset $\mathcal{D}$. When splitting a node within tree $m$, instead of evaluating all $d$ available features, a random subset of features $\mathcal{M} \subset \{1, 2, \dots, d\}$ (where $|\mathcal{M}| \approx \sqrt{d}$) is sampled to select the optimal splitting criterion based on Gini Impurity or Information Gain. The individual decision tree prediction for a transaction vector $\mathbf{x}$ is denoted as $T_m(\mathbf{x}) \in \{0, 1\}$.

The final classification decision is determined via a majority voting scheme across all constituent trees:
\begin{equation}
\hat{y}_{\text{RF}} = \text{mode}\left\{ T_1(\mathbf{x}), T_2(\mathbf{x}), \dots, T_M(\mathbf{x}) \right\}
\end{equation}
Alternatively, continuous risk probabilities are computed by averaging the predicted class probabilities across the ensemble:
\begin{equation}
P(y=1|\mathbf{x}) = \frac{1}{M} \sum_{m=1}^{M} P_m(y=1|\mathbf{x})
\end{equation}
By aggregating decentralized, low-bias but high-variance models, Random Forest substantially reduces global variance and prevents overfitting, making it a highly resilient tabular classifier.

\subsubsection{XGBoost (Extreme Gradient Boosting)}\cite{XGB}
Unlike the parallel assembly of Random Forest, XGBoost (Extreme Gradient Boosting) constructs an additive ensemble sequentially. It minimizes a regularized objective function by fitting weak base learners (typically regression trees) to the negative gradients (pseudo-residuals) of the loss function computed in preceding iterations.

At iteration $t$, the prediction $\hat{y}_i^{(t)}$ for sample $i$ is updated from the prior prediction $\hat{y}_i^{(t-1)}$ by adding a new tree function $f_t(\mathbf{x}_i) \in \mathcal{F}$, where $\mathcal{F}$ represents the space of all possible Classification and Regression Trees (CARTs):
\begin{equation}
\hat{y}_i^{(t)} = \hat{y}_i^{(t-1)} + f_t(\mathbf{x}_i)
\end{equation}
The global objective function $\mathcal{O}^{(t)}$ to be minimized at step $t$ incorporates a differentiable loss function $\ell$ alongside a structural regularization term $\Omega$ that penalizes tree complexity to prevent overfitting:
\begin{equation}
\mathcal{O}^{(t)} = \sum_{i=1}^{B} \ell\left(y_i, \hat{y}_i^{(t-1)} + f_t(\mathbf{x}_i)\right) + \Omega(f_t)
\end{equation}
where the regularizer over a tree with $T$ leaves and leaf weights $w$ is defined as:
\begin{equation}
\Omega(f_t) = \gamma T + \frac{1}{2} \lambda \sum_{j=1}^{T} w_j^2
\end{equation}
Taking a second-order Taylor expansion of the objective function allows XGBoost to utilize custom optimization objectives efficiently. The objective simplifies to:
\begin{equation}
\mathcal{O}^{(t)} \approx \sum_{i=1}^{B} \left[ \ell(y_i, \hat{y}_i^{(t-1)}) + g_i f_t(\mathbf{x}_i) + \frac{1}{2} h_i f_t^2(\mathbf{x}_i) \right] + \Omega(f_t)
\end{equation}
where $g_i = \partial_{\hat{y}^{(t-1)}} \ell(y_i, \hat{y}_i^{(t-1)})$ and $h_i = \partial^2_{\hat{y}^{(t-1)}} \ell(y_i, \hat{y}_i^{(t-1)})$ represent the first-order (gradient) and second-order (hessian) partial derivatives of the loss function, respectively. XGBoost provides state-of-the-art predictive performance for static tabular logs by leveraging parallel tree boosting, sparse-aware splitting, and localized hardware cache optimization.
\subsubsection{The TransformerClassifier Architecture Overview}
Unlike traditional classifiers that treat transactions as static and independent tabular vectors, the proposed \texttt{TransformerClassifier} architecture maps transactional features into sequential token representations to capture complex feature-to-feature interactions. \subsubsection{The Mathematics of Self-Attention (Transformer Core)}
While Logistic Regression, Random Forest, and XGBoost evaluate transactions as static, independent tabular vectors, the Transformer architecture models mixed categorical and numerical features as structured sequential tokens of a single transaction. Let an input sequence representing the transactional attributes (specifically the execution hour, merchant category, and log-transformed amount) be compiled into a matrix $\mathbf{X} \in \mathbb{R}^{S \times d_{\text{model}}}$, where $S = 3$ defines the absolute sequence length (number of input features) and $d_{\text{model}}$ represents the internal latent embedding dimension.

To evaluate the relative contextual correlation between different transaction events within the temporal window, the sequence is mapped into three distinct inner vector \cite{transformer}: Queries ($\mathbf{Q}$), Keys ($\mathbf{K}$), and Values ($\mathbf{V}$). This projection is executed using three distinct trained linear weight layers $\mathbf{W}_Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$, $\mathbf{W}_K \in \mathbb{R}^{d_{\text{model}} \times d_k}$, and $\mathbf{W}_V \in \mathbb{R}^{d_{\text{model}} \times d_v}$:
\begin{equation}
\mathbf{Q} = \mathbf{X}\mathbf{W}_Q, \quad \mathbf{K} = \mathbf{X}\mathbf{W}_K, \quad \mathbf{V} = \mathbf{X}\mathbf{W}_V
\end{equation}
The scaled dot-product attention computes the matrix of attention weights by calculating the inner product between all pairs of queries and keys. To prevent the gradients from vanishing or exploding when the projection dimension $d_k$ grows large, the dot products are scaled by a factor of $1 / \sqrt{d_k}$ before applying a softmax activation layer:
\begin{equation}
\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}}\right)\mathbf{V}
\end{equation}

\subsubsection{Multi-Head Attention Assembly}
To allow the network to simultaneously map relationships across different transaction dimensions (e.g., correlating unusual hours with unexpected spending amounts), a Multi-Head Attention architecture is utilized. Let $h$ define the number of parallel attention heads. Each individual head $i$ learns independent projection mappings:
\begin{align}
\text{head}_i &= \text{Attention}(\mathbf{Q}\mathbf{W}_Q^i, \mathbf{K}\mathbf{W}_K^i, \mathbf{V}\mathbf{W}_V^i) \\
\text{MultiHead}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) &= \text{Concat}(\text{head}_1, \text{head}_2, \dots, \text{head}_h)\mathbf{W}^O
\end{align}
where $\mathbf{W}^O \in \mathbb{R}^{h d_v \times d_{\text{model}}}$ represents the linear output matrix layer that merges all attention perspectives back into a single vector space.

\subsubsection{Positional Encoding Formulations}
Because the pure self-attention mechanism relies entirely on matrix dot-products, it is completely invariant to the permutations of rows within the input matrix $\mathbf{X}$. To preserve the distinct structural role of each feature token within the sequence (distinguishing between execution hour, merchant category, and transaction amount), fixed sinusoidal Positional Encodings ($\mathbf{PE}$) are superimposed directly onto the base feature embeddings. Let $pos \in \{0, 1, 2\}$ represent the explicit chronological position of a feature token within the transaction representation sequence, and let $i$ represent the internal dimension channel:
\begin{align}
\mathbf{PE}(pos, 2i) &= \sin\left(\frac{pos}{10000^{\frac{2i}{d_{\text{model}}}}}\right) \\
\mathbf{PE}(pos, 2i+1) &= \cos\left(\frac{pos}{10000^{\frac{2i}{d_{\text{model}}}}}\right)
\end{align}
This technique allows the network to learn relative positions, as any linear shift $\mathbf{PE}_{pos+k}$ can be expressed as a linear function of $\mathbf{PE}_{pos}$.

\subsubsection{Layer Normalization and Residual Structures}
To stabilize the deep network architecture and ensure clean gradient flow through multiple layers of multi-head attention blocks, each sub-layer incorporates a residual skip-connection followed by a Layer Normalization (\texttt{LayerNorm}) block. This structural operation is formalized as follows:
\begin{equation}
\text{SubLayer}_{\text{out}} = \text{LayerNorm}(\mathbf{X} + \text{SubLayer}(\mathbf{X}))
\end{equation}
The \texttt{LayerNorm} operation computes the precise mean $\mu$ and variance $\sigma^2$ across all feature channels for each individual sample, applying two learnable parameters—gain ($\gamma$) and bias ($\beta$)—to rescale the internal activations:
\begin{equation}
\text{LayerNorm}(\mathbf{z}) = \frac{\mathbf{z} - \mu}{\sqrt{\sigma^2 + \epsilon}} \odot \gamma + \beta
\end{equation}
where $\epsilon$ is a tiny positive regularization constant used to prevent division-by-zero errors.



\subsection{Loss Functions and Optimization Mechanics}
The optimization of the parametric networks (Logistic Regression and Transformer)\cite{Logistic Regression and Transformer} is guided by the Binary Cross-Entropy with Logits loss function, penalizing structural misclassifications between the predicted fraud logit and the ground-truth target binary flag $y \in \{0, 1\}$:
\begin{equation}
\mathcal{L}_{\text{BCE}} = - \frac{1}{B} \sum_{j=1}^{B} \left[ y_j \cdot \log(\sigma(\hat{y}_j)) + (1 - y_j) \cdot \log(1 - \sigma(\hat{y}_j)) \right]
\end{equation}
where $B$ defines the mini-batch size, $\hat{y}_j$ is the raw unnormalized logit outputted by the final linear feed-forward layer, and $\sigma(z) = 1 / (1 + e^{-z})$ represents the standard sigmoid function mapping logits into explicit continuous risk probabilities.

For the Transformer model, gradient updates are calculated using the \textbf{AdamW} optimizer, which decouples traditional weight decay from the moving averages of the first and second moments of the gradients. For XGBoost, numerical optimization is achieved natively via a custom gradient descent algorithm across tree structures as detailed in the objective function derivations above.

\subsection{Data Preprocessing and Exploratory Data Analysis}

\subsubsection{Data Ingestion and Feature Cleanliness} 
The computational pipeline begins with the ingestion of separate training (\texttt{fraudTrain.csv}) and testing (\texttt{fraudTest.csv}) data structures. To prevent target overfitting and focus the model on highly generalizable risk indicators, the feature space is strictly constrained. High-cardinality demographic, personal, and spatial identifiers—specifically account card numbers (\texttt{cc\_num}), transaction reference markers (\texttt{trans\_num}), customer names (\texttt{first}, \texttt{last}), street addresses, cities, states, occupations (\texttt{job}), birth dates (\texttt{dob}), system timestamps (\texttt{unix\_time}), and raw geographical coordinates (\texttt{lat}, \texttt{long}, \texttt{merch\_lat}, \texttt{merch\_long})—are systematically dropped. The final modeling space is focused exclusively on three primary transactional inputs: the execution hour (\texttt{trans\_hour}), merchant category (\texttt{category}), and log-scaled amount (\texttt{amt\_log}).

\subsubsection{Feature Engineering and Non-linear Transformations} 
To maximize the predictive capacity of the temporal and monetary parameters, advanced feature transformation techniques are applied: 
\begin{enumerate}
    \item \textbf{Temporal Decomposition}: The raw string timestamp \texttt{trans\_date\_trans\_time} is dynamically parsed into an explicit DateTime object. From this temporal foundation, localized discrete time-based dimensions are extracted for analysis, including the transaction hour of the day (\texttt{trans\_hour}), calendar day (\texttt{trans\_day}), and chronological month (\texttt{trans\_month}). While all three dimensions are utilized during exploratory visualization to diagnose fraud temporal distributions, only the diurnal hour (\texttt{trans\_hour}) is selected for the model's feature space, with calendar days and seasonal months excluded to prevent model memorization and overfitting. 
    \item \textbf{Mathematical Log-Transformation}: Preliminary empirical distribution checking of the financial transaction amount (\texttt{amt}) reveals an extreme right-skewed profile.  To compress the dynamic range and minimize gradient variances during neural network training, a log-transformation with a leveling unity offset is applied to formulate the engineered feature \texttt{amt\_log}: 
    \begin{equation}
        \text{amt\_log} = \ln(\text{amt} + 1)
    \end{equation}
\end{enumerate}

\subsubsection{Class Imbalance Mitigation via Random Under-Sampling}
A primary challenge in digital transaction analysis is the severe class imbalance, where fraudulent transactions constitute only $0.5789\%$ of the global dataset (with $2,145$ fraud cases out of $555,719$ sequences in the test partition). Training deep sequence models directly on such skewed data would cause the self-attention heads to optimize for the majority class, leading to severe false-negative leaks. To mitigate this risk, offline Random Under-Sampling (RUS) is applied to the training dataset partition ($\mathcal{D}_{\text{train}}$) using the \texttt{imblearn} library's \texttt{RandomUnderSampler} with a fixed seed (\texttt{random\_state = 42}).

Let $\mathcal{D}_{\text{train}} = \mathcal{D}_{\text{train}, 0} \cup \mathcal{D}_{\text{train}, 1}$ denote the training set, where $\mathcal{D}_{\text{train}, 0}$ represents the majority set of legitimate transactions and $\mathcal{D}_{\text{train}, 1}$ represents the minority set of fraudulent transactions. The under-sampling operator selects a subset $\mathcal{D}^*_{\text{train}, 0} \subset \mathcal{D}_{\text{train}, 0}$ such that:
\begin{equation}
|\mathcal{D}^*_{\text{train}, 0}| = |\mathcal{D}_{\text{train}, 1}|
\end{equation}
The resulting balanced training partition $\mathcal{D}^*_{\text{train}} = \mathcal{D}^*_{\text{train}, 0} \cup \mathcal{D}_{\text{train}, 1}$ achieves a strict $1:1$ class ratio ($50.0\%$ fraud and $50.0\%$ clean), forcing the multi-head self-attention mechanisms to learn the structural characteristics of both classes equally. During evaluation, the models are tested on the pristine, highly skewed test partition ($\mathcal{D}_{\text{test}}$) to validate their generalization performance in a realistic production-grade class distribution.

\subsection{Database Schema and Behavioral Intelligence Framework}

The proposed fraud detection system employs a normalized relational database that not only supports traditional e-commerce operations but also serves as the foundation of the behavioral intelligence architecture. The database is designed to separate transactional data from analytical user profiles, allowing both operational efficiency and real-time risk evaluation.

\subsubsection{Relational Data Modeling}

The database consists of six primary entities: \texttt{Users}, \texttt{Products}, \texttt{Purchases}, \texttt{PurchaseItems}, \texttt{Transactions}, and \texttt{UserRiskProfile}. Their relationships are illustrated in the system architecture and summarized as follows:

\begin{itemize}
    \item \textbf{Users}: Stores customer information, authentication credentials, account creation time, verification status, and other attributes required by the Trust Engine.
    
    \item \textbf{Products}: Maintains product information including category, price, and inventory, which are required during purchase processing and fraud feature extraction.
    
    \item \textbf{Purchases} and \textbf{PurchaseItems}: Represent customer orders and purchased products. These entities provide historical purchasing behavior that contributes to trust score calculation.
    
    \item \textbf{Transactions}: Records every completed payment together with its behavioral features, including transaction amount, category, transaction time, fraud probability predicted by the ONNX model, and the final fraud classification.
    
    \item \textbf{UserRiskProfile}: Stores consolidated analytical information generated by the behavioral engines, including the current Risk Score, Trust Score, corresponding risk levels, and profile update timestamp.
\end{itemize}

By separating immutable transaction logs from aggregated user profiles, the system minimizes redundancy while enabling efficient analytical queries for real-time fraud detection.

\subsubsection{Behavioral Feature Engineering}

Unlike conventional fraud detection systems that rely solely on individual transaction attributes, the proposed framework combines transactional features with long-term behavioral indicators.

The feature engineering pipeline consists of three major stages:

\begin{enumerate}
    \item \textbf{Transaction Feature Extraction}: Basic features required by the Transformer classifier are extracted directly from the transaction context, including transaction amount, product category, transaction hour.

    \item \textbf{Behavioral Feature Aggregation}: Historical user statistics are dynamically computed from the relational database. These include fraud rate, average fraud probability, night transaction rate, transaction frequency, purchase success rate, account age, verification status, and purchase activity. These behavioral features are subsequently utilized by the Risk Engine and Trust Engine.

    \item \textbf{Feature Normalization}: Continuous numerical attributes are normalized before inference. In particular, transaction amount is transformed using logarithmic scaling:
    
    \begin{equation}
        x_{\mathrm{amt}}=\ln(\mathrm{amount}+1)
    \end{equation}

    This transformation reduces the influence of extremely large transactions while preserving relative differences among normal purchases.
\end{enumerate}

The normalized feature vector is then serialized into the input representation required by the Transformer-based ONNX model:

\begin{equation}
\mathbf{X}_{\mathrm{input}}
=
\mathrm{Concat}
\left(
\mathrm{Hour},
\mathrm{Category},
x_{\mathrm{amt}}
\right)
\end{equation}

The inference result is the fraud probability of the current transaction.

\subsubsection{Behavioral Intelligence Layer}

The proposed fraud detection framework extends conventional transaction-level analysis by incorporating two complementary behavioral evaluation modules: the \textbf{Risk Engine} and the \textbf{Trust Engine}. While the Risk Engine evaluates the short-term probability of fraudulent behavior based on recent transaction patterns, the Trust Engine measures the long-term reliability of a customer through historical purchasing behavior and account characteristics.

Separating these two dimensions enables the system to distinguish between temporary behavioral anomalies and the overall credibility of a customer. Consequently, legitimate users with strong historical reputations are less likely to be incorrectly classified as fraudulent, thereby reducing false positive decisions.

\subsubsection{Risk Engine}

The Risk Engine computes a quantitative Risk Score, denoted as $\mathcal{R}\in[0,100]$, by aggregating four behavioral indicators extracted from historical transaction records.

The overall Risk Score is defined as

\begin{equation}
\mathcal{R}
=
0.40F_r
+
0.30P_{avg}
+
0.20N_r
+
0.10F_q
\end{equation}

where

\begin{itemize}

\item \textbf{Fraud Rate ($F_r$ -- 40\%)} measures the proportion of transactions previously classified as fraudulent:

\begin{equation}
F_r
=
\frac{N_{\mathrm{fraud}}}
{N_{\mathrm{transaction}}}
\times100
\end{equation}

\item \textbf{Average Fraud Probability ($P_{avg}$ -- 30\%)} represents the average fraud probability predicted by the Transformer-based ONNX model:

\begin{equation}
P_{avg}
=
\frac{
\sum p_i
}
{
N_{\mathrm{transaction}}
}
\times100
\end{equation}

where $p_i$ denotes the fraud probability of transaction $i$.

\item \textbf{Night Transaction Rate ($N_r$ -- 20\%)} evaluates the percentage of transactions performed between 22:00 and 06:00:

\begin{equation}
N_r
=
\frac{
N_{\mathrm{night}}
}
{
N_{\mathrm{transaction}}
}
\times100
\end{equation}

\item \textbf{Transaction Frequency ($F_q$ -- 10\%)} reflects the customer's transaction activity level. To prevent highly active accounts from disproportionately dominating the score, the frequency indicator is capped and normalized:
\begin{equation}
F_q = \min(N_{\text{transaction}}, 100)
\end{equation}

\end{itemize}

The resulting score is classified into three operational levels:

\begin{center}
\begin{tabular}{cc}
\hline
Risk Score & Risk Level\\
\hline
0--30 & LOW\\
31--70 & MEDIUM\\
71--100 & HIGH\\
\hline
\end{tabular}
\end{center}

\subsubsection{Trust Engine}

Unlike the Risk Engine, which focuses on recent behavioral anomalies, the Trust Engine evaluates the long-term credibility of a customer. The Trust Score, denoted as $\mathcal{T}\in[0,100]$, is derived from historical account activity and purchasing behavior.

The Trust Score is computed as

\begin{equation}
\mathcal{T}
=
0.20A
+
0.30S
+
0.30H
+
0.10V
+
0.10P
\end{equation}

where

\begin{itemize}

\item \textbf{Account Age ($A$ -- 20\%)} reflects the stability of the customer's account. Older accounts receive higher trust values.

\item \textbf{Purchase Success Rate ($S$ -- 30\%)} measures the proportion of successfully completed purchases relative to the total number of purchase requests.

\item \textbf{Fraud History ($H$ -- 30\%)} evaluates the customer's historical fraud behavior. Users with fewer confirmed fraudulent transactions receive higher scores.

\item \textbf{Verification Score ($V$ -- 10\%)} reflects account verification status, including verified email and verified phone number.

\item \textbf{Purchase Activity ($P$ -- 10\%)} measures long-term purchasing activity. Customers with consistent purchasing histories receive higher reputation scores than newly created or inactive accounts.

\end{itemize}

The Trust Score is categorized into four reputation levels:

\begin{center}
\begin{tabular}{cc}
\hline
Trust Score & Trust Level\\
\hline
90--100 & EXCELLENT\\
75--89 & GOOD\\
50--74 & AVERAGE\\
0--49 & POOR\\
\hline
\end{tabular}
\end{center}

The Risk Engine and Trust Engine jointly provide a comprehensive behavioral profile of each customer. While the Risk Engine captures immediate transaction risk, the Trust Engine reflects long-term customer reliability. Together, these complementary assessments enable the system to support adaptive fraud mitigation strategies, improve decision quality, and reduce false positive classifications without compromising security.
\subsubsection{Decision Engine}

Although the Transformer-based classifier provides the fraud probability of an individual transaction, relying solely on this prediction may lead to false positive decisions, particularly for long-term customers with strong purchasing histories. To address this limitation, the proposed framework introduces a rule-based \textbf{Decision Engine} that combines the outputs of the Risk Engine and the Trust Engine to determine the appropriate response for each transaction.

Instead of treating all high-risk transactions identically, the Decision Engine considers both the customer's current behavioral risk and accumulated reputation. The final decision is selected from four operational actions:

\begin{itemize}
    \item \textbf{APPROVE}: The transaction is accepted immediately.
    \item \textbf{REVIEW}: The transaction is temporarily held for further inspection.
    \item \textbf{BLOCK}: The transaction is rejected due to excessive fraud risk.
\end{itemize}

The decision policy is implemented as a deterministic decision matrix shown in Table~\ref{tab:decision_engine}.

\begin{table}[h]
\centering
\caption{Decision matrix combining Risk Level and Trust Level}
\label{tab:decision_engine}
\begin{tabular}{lccc}
\hline
\textbf{Risk Level} & \textbf{Poor/Average} & \textbf{Good} & \textbf{Excellent} \\
\hline
LOW    & APPROVE & APPROVE & APPROVE \\
MEDIUM & REVIEW & REVIEW & APPROVE \\
HIGH   & BLOCK & REVIEW & REVIEW \\
\hline
\end{tabular}
\end{table}

The rationale behind this strategy is that Risk Score reflects short-term behavioral anomalies, whereas Trust Score represents long-term customer credibility. Consequently, customers with consistently trustworthy histories are not immediately blocked when a single transaction appears suspicious. Instead, the system applies adaptive verification mechanisms that preserve both security and user experience.

For example, a transaction classified as \texttt{HIGH} risk but associated with an \texttt{EXCELLENT} trust profile is redirected to additional verification rather than being rejected outright. Conversely, transactions exhibiting both high risk and poor reputation are immediately blocked to minimize potential financial losses.

This hybrid decision mechanism effectively balances fraud prevention and customer convenience by reducing unnecessary transaction rejections while maintaining strong protection against malicious activities.

The proposed system follows a layered architecture that integrates deep learning-based fraud prediction with a behavioral intelligence framework. Each purchase request is processed through multiple analytical stages, enabling the system to evaluate both the immediate fraud risk of a transaction and the long-term credibility of the customer before producing the final decision.

\subsection{Model Training Diagnostics and Convergence Tracks}

To rigorously evaluate the predictive efficacy of the proposed Transformer model against traditional machine learning baselines, empirical benchmarks were established using Logistic Regression, Random Forest, and XGBoost. A critical challenge embedded within the credit card fraud dataset is the presence of an extreme class imbalance, where the negative class (Non-Fraudulent, $N = 553,574$) completely dominates the positive minority class (Fraudulent, $N = 2,145$). \begin{itemize}
    \item \textbf{Linear and Ensemble Baselines Diagnostics}:

    The single Decision Tree model constructs orthogonal non-linear splitting boundaries, achieving a high recall of $0.9688$ but suffering from high structural variance, which results in a high number of False Positives ($13,831$), a precision of $0.1306$, and an $F_1$-score of $0.2302$ with an ROC AUC of $0.9671$.

    \item \textbf{Transformer Behavioral Analysis}: 
    As shown in the classification matrix, the sequence-centric Transformer model demonstrates a profound capacity to isolate fraudulent anomalies, securing an overall accuracy tier of $0.97$. Focusing specifically on the minority class metrics, the Transformer captures the fraudulent vectors with a remarkably high sensitivity rate ($\text{Recall} = 0.9688$) while anchoring a superior precision envelope of $0.13$ compared to the baselines, culminating in an improved minority $F_1$-score of $0.2302$. Furthermore, it achieves a strong Precision-Recall AUC (PR-AUC) of $0.8026$, demonstrating high robust classification capability.

    \item \textbf{Architectural Trade-offs}: 
    While the traditional ensemble frameworks leverage decision trees to optimize their split-point thresholds, the Transformer framework operates on an entirely different structural premise. By utilizing self-attention mechanics over transactional history windows, the Transformer model maintains a comparable and competitive recall profile ($0.9688$). This high recall capacity is paramount in institutional fraud detection operations, as minimizing false negatives (undetected financial crimes) takes technical precedence over the minor inflation of false positives. The capability of the Transformer to yield a weighted average $F_1$-score of $0.98$ and a macro average $F_1$-score of $0.61$ across the entire evaluation horizon ($555,719$ sequences) confirms its operational reliability and high stability under extreme data skewness.
\end{itemize}

\section{Discussion, Limitations, and Future Work}
This section provides a critical appraisal of the developed \texttt{TransformerClassifier} and its full-stack microservice orchestration. While the empirical results demonstrate exceptional predictive recall and sub-millisecond local backend execution latency, an enterprise-grade transition requires a transparent acknowledgment of existing algorithmic and architectural constraints. Consequently, the following discourse evaluates current operational limitations and outlines a strategic roadmap for future system development.

\subsection{System Limitations and Critical Appraisal}
Despite achieving a high anomaly detection rate, the current architecture exhibits critical technical vulnerabilities across two main dimensions:

\begin{enumerate}
    \item \textbf{Data Information Attrition via Random Under-Sampling}: To mitigate the extreme class skew inherent in the financial logs, a strict Random Under-Sampling (RUS) technique was enforced during the offline training stage. Although this successfully balances the class distribution and forces the multi-head self-attention mechanisms to prioritize the minority \texttt{Fraud} class, it introduces significant information attrition. By randomly dropping over $90\%$ of the legitimate transaction population ($df_{\text{train}}$), the model loses exposure to high-frequency structural variants of normal customer shopping behavior. This systemic data deprivation directly causes the high False Positive Rate (low localized precision of $13.00\%$) documented during empirical validation.
    
    \item \textbf{Stale Optimization Blocks and Absence of Online Adaptive Learning}: The current inference engine operating within the Spring Boot domain utilizes a static frozen weight architecture (\texttt{model.onnx}). Financial fraud topologies are inherently dynamic, characterized by rapid "concept drift" where adversarial actors continuously alter their attack vectors. Because the embedded ONNX setup lacks an online adaptive learning pipeline, the model cannot update its structural parameters incrementally based on real-time transaction feedback. Remediation requires an expensive offline re-training cycle, creating a window of vulnerability against novel, zero-day anomalous signatures.
\end{enumerate}

\subsection{Strategic Roadmap and Evolutionary Architecture}
To overcome the aforementioned algorithmic limits and elevate the backend's structural scalability to support massive corporate throughput, three strategic architectural upgrades are proposed for the next deployment phase:

\begin{enumerate}
    \item \textbf{Advanced Architectural Decoupling via Microservice Atomicity}: The core backend domain presently concentrates extensive business rules within the unified \texttt{PurchaseService}. To maximize microservice horizontal scaling, this structure will be decomposed into decoupled atomic services. An independent \texttt{InventoryService} will handle state synchronization and non-blocking inventory deductions, while a dedicated \texttt{PricingService} will isolate transactional billing computations. The primary \texttt{PurchaseService} will evolve into a lightweight event orchestrator, preventing thread starvation and reducing horizontal microservice latency.
    
    \item \textbf{Structural Hiding via the PredictionFacade Structural Pattern}: Within the existing operational flow, the \texttt{TransactionService} maintains a direct dependency on the internal data structures of the \texttt{FraudDetectionService}. To enforce strict encapsulation and improve modularity, a \texttt{PredictionFacade} architecture will be implemented. This interface layer will mask the complexities of tensor packaging, buffer pooling, and native ONNX resource lifecycle management. This ensures that any downstream modification to the AI runtime (such as transitioning from ONNX back to a distributed cloud endpoint) can be executed without impacting core e-commerce accounting frameworks.
    
    \item \textbf{Synthesized Oversampling and Cost-Sensitive Loss Functions}: To resolve the information loss driven by raw under-sampling, future iterations will shift toward sophisticated data-generation mechanics. Synthetic Minority Over-sampling Techniques (SMOTE) or ADASYN will be integrated to construct continuous synthetic fraud topologies, preserving the entire pristine dataset of legitimate entries. Furthermore, the objective function will transition from standard Binary Cross-Entropy (\texttt{BCEWithLogitsLoss}) to an asymmetric \textbf{Focal Loss} or a cost-sensitive penalty matrix:
    \begin{equation}
        \mathcal{L}_{\text{Focal}} = -\alpha_t (1 - p_t)^\gamma \log(p_t)
    \end{equation}
    where $\gamma$ represents the dynamic hyperparameter managing down-weighting for easy-to-classify legitimate logs. This mathematical transition will suppress the gradient impact of ubiquitous clean transactions, allowing the system to achieve elite precision constraints without degrading its current $96.88\%$ anomaly sensitivity.
    
    \item \textbf{Integration of Long-Term Behavioral Features in Deep Modeling}: Currently, the simplified deep-learning classification head operates solely on static transaction-level attributes (hour, category, and log-amount). In the next evolution, dynamic, long-term historical user statistics (such as historical fraud rates and transaction frequencies) will be integrated directly into the input embeddings of the \texttt{TransformerClassifier}. This will allow the self-attention heads to map correlations across both static feature categories and dynamic historical behaviors simultaneously.
\end{enumerate}

\section{Conclusion}
Throughout this internship project, a comprehensive deep learning framework and operational software ecosystem were successfully engineered to address the challenges of automated financial fraud detection. By exploiting the multi-head self-attention mechanisms of the Transformer architecture, the system moves beyond traditional isolated data entry checks, capturing non-linear sequential relationships across transaction histories. 

Empirical validation on a large-scale dataset confirms that the trained model achieves an exceptional anomaly sensitivity rate, successfully catching approximately $97\%$ of all fraudulent checkout attempts ($\text{Recall} = 0.9688$). Furthermore, the encapsulation of the optimized network weights inside a local, in-memory Spring Boot ONNX Runtime engine establishes a blueprint for zero-network-overhead enterprise architectures, fulfilling the micro-latency requirements of online banking infrastructure. While limitations regarding the high false positive rate remain due to random under-sampling constraints ($Precision = 0.13$), the proposed evolutionary path—incorporating cost-sensitive focal loss boundaries and decentralized microservice layers—establishes a robust roadmap for production readiness. Ultimately, this internship successfully bridges advanced machine learning research with enterprise-tier software engineering, contributing a scalable solution toward securing the modern digital financial economy.
